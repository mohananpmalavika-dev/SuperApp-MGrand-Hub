   - Track progress

   - Use for quick revision

   - Track mastery

## 💾 Storage Strategy

### What Goes in MongoDB:
- Course metadata only (~5 KB per subject)
- Lesson titles and structure
- Google Drive file IDs
- User progress data

### What Stays in Google Drive:
- Complete JSON files with all content
- Audio/Video URLs embedded in JSON
- Notes data in JSON (generate on-demand)

### What's Generated On-Demand:
- PDF notes (from JSON data)
- DOCX notes (from JSON data)
- TXT notes (from JSON data)

**Result:** MongoDB stays under 500MB, unlimited content in Drive!

## 📤 After Generation

### Step 1: Upload to Google Drive
```bash
npm run upload:drive
```
Drag & drop the 4 JSON files

### Step 2: Make Files Public
- Right-click each file → Share
- Change to "Anyone with the link"
- Copy file IDs

### Step 3: Setup Drive System
```bash
npm run setup:drive-system
```
Update with file IDs

### Step 4: Test Access
```bash
curl http://localhost:3013/api/education/drive/content/FILE_ID
```
